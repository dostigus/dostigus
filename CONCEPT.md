# 🪿 Достигусь / Dostigus — Концепт

**Self-hosted, open-source personal media tracker with AI recommendations and achievements.**

Track books, music, movies, games — all in one place. Your AI Goose knows your taste.

---

## Проблема

Каждый из нас потребляет контент: читает книги, слушает музыку, смотрит фильмы, играет в игры. И каждый сталкивается с одними и теми же проблемами:

- **Разрозненность.** Книги в Goodreads, музыка в Last.fm, фильмы в Letterboxd или Kinorium, игры в Steam. Нет единого места.
- **Потеря контекста.** Не помнишь когда читал книгу, что о ней думал, в каком настроении слушал альбом.
- **Тупые рекомендации.** Алгоритмы стриминговых сервисов оптимизируют engagement, а не твой вкус. Никто не смотрит на всю картину целиком.
- **Нет мотивации.** Прочитал 50 книг за год — и что? Никакого признания, никакого прогресса.

## Решение

**Достигусь** — это персональная медиатека с AI-гусём и системой достижений.

### Три слоя

**1. Каталог (данные)**
Единое хранилище всего что ты читал, слушал, смотрел, играл. Каждая запись содержит: название, метаданные (автор, жанр, год — подтягиваются автоматически из открытых API), твоя оценка, заметки, теги, дата добавления / потребления. Данные приходят из провайдеров — внешних сервисов и ручного ввода.

**2. AI-гусь (мозг)**
AI видит всю твою историю целиком — кросс-медийно. Он может:
- Рекомендовать следующую книгу/альбом/фильм с объяснением почему
- Замечать паттерны в твоих вкусах
- Строить кросс-медийные связи ("тебе понравился Солярис Лема — послушай этот ambient альбом")
- Отвечать на вопросы о твоей коллекции ("какие книги я читал прошлым летом?")

**3. Достижения (геймификация)**
Два типа ачивок:
- **Статические** — правила в коде: "прочитал 10 книг", "послушал альбом каждый день неделю", "попробовал 5 новых жанров". Предсказуемые, работают оффлайн, мотивируют.
- **AI-генерируемые** — персональные, которые замечает AI: "ты прошёл путь от попсы до прог-рока за полгода", "три месяца подряд читаешь только японских авторов". Уникальные для каждого пользователя.

## Провайдеры данных

Плагинная архитектура. Каждый медиа-тип получает данные из внешних источников через провайдеры.

### Музыка
- **Мелотека** — коллекция, скроблы, оценки (кастомный API)
- **ListenBrainz** — открытый API, скроблы
- **Spotify** — API для импорта плейлистов и истории
- **Last.fm** — API скроблов
- **Ручной ввод** — с обогащением через MusicBrainz API

### Книги
- **Open Library API** — метаданные, обложки, описания (бесплатный, открытый)
- **Google Books API** — поиск и метаданные
- **Goodreads CSV** — импорт существующей коллекции
- **Ручной ввод** — поиск по названию, автоподтягивание метаданных

### Фильмы и сериалы
- **TMDB API** — метаданные, постеры (бесплатный, открытый)
- **Kinorium** — импорт (если есть экспорт)
- **Trakt.tv** — открытый API, скроблы
- **Letterboxd CSV** — импорт
- **Ручной ввод**

### Игры
- **IGDB / Twitch API** — метаданные
- **Steam API** — библиотека и время в игре
- **RAWG API** — метаданные
- **Ручной ввод**

### Расширяемость

Провайдер — это интерфейс: `search()`, `getMetadata()`, `importCollection()`, `sync()`. Любой может написать свой провайдер и подключить.

## AI-гусь

### Как это работает

AI-слой не привязан к конкретному LLM. Поддерживаемые бэкенды:
- **Claude API** (Anthropic) — рекомендуется
- **OpenAI API**
- **Ollama** — для полностью локальной работы, оффлайн
- **Любой OpenAI-совместимый API**

### Что умеет

- **Рекомендации.** "Что послушать?" — AI получает контекст: топ-альбомы по оценкам, последние скроблы, любимые жанры, текущее настроение (опционально). Возвращает рекомендации с объяснениями. То же для книг, фильмов, игр.
- **Кросс-медийные связи.** AI видит всё: "тебе понравился фильм Бегущий по лезвию и книга Нейромант — вот synth-wave альбом который тебе зайдёт".
- **Инсайты.** "Ты последний месяц читаешь только нон-фикшн — вот хорошая фантастика для баланса". Паттерны которые сам не замечаешь.
- **Вопросы к коллекции.** "Какие книги я прочитал в 2024?", "Сколько часов музыки я послушал?", "Какой мой самый продуктивный месяц по книгам?"

### Промпт-архитектура

Структурированный контекст для каждого запроса:
- Профиль пользователя (любимые жанры, средние оценки)
- Релевантная часть коллекции (не вся — а подборка по теме запроса)
- История рекомендаций (чтобы не повторяться)
- Системный промпт с ролью AI-гуся

## Система достижений

### Философия

Как в WoW — ачивки видимые, собираемые, с прогресс-баром. Но для реальной жизни.

### Категории

**По медиа-типу:**
- Книги: "Книжный червь" (10 книг), "Книголюб" (50 книг), "Книжный маньяк" (100 книг)
- Музыка: "Меломан" (50 альбомов), "Диджей" (послушал 10 жанров), "Vinyl Soul" (100 альбомов)
- Фильмы: "Кинокритик" (50 фильмов), "Марафонец" (5 фильмов за день)
- Игры: "Геймер" (10 игр), "Ачивка-хантер" (50 игр)

**Кросс-медийные:**
- "Мультимедиа" — добавил записи во всех 4 категориях
- "Культурный код" — 100 записей суммарно
- "Энциклопедист" — 500 записей суммарно

**По паттернам:**
- "Стрик" — добавлял что-то каждый день неделю
- "Исследователь" — попробовал 10 новых жанров за месяц
- "Ностальгик" — перечитал/переслушал что-то из коллекции
- "Рецензент" — оставил заметки к 20 записям

**AI-генерируемые (уникальные):**
- Персональные ачивки которые AI создаёт на основе паттернов
- Иконка, название, описание — всё генерирует AI
- Пример: "Скандинавский след" — прочитал 5 скандинавских авторов за год

### Прогресс

Каждая ачивка может иметь уровни (бронза/серебро/золото) и прогресс-бар. Профиль пользователя показывает набранные ачивки как витрину.

## Социальный слой (cloud)

> Социальные фичи доступны на dostigus.ru. В self-hosted версии — только персональный трекер.

### Публичный профиль
- Коллекция (или её часть — настраиваемая приватность)
- Витрина ачивок
- Статистика (сколько прочитал/послушал/посмотрел)

### Общие данные
- "247 человек прочитали эту книгу, средняя оценка 8.2"
- Комментарии и мини-рецензии
- "Люди с похожим вкусом также читали..."

### Активность
- Лента: "Ник прочитал Солярис и поставил 9/10"
- Возможность подписываться на друзей
- Клубы / группы (книжный клуб, музыкальный клуб)

## Маскот

**Гусь-достигатор** — целеустремлённый гусь, который всё достигает. Мемный, узнаваемый, с характером. Появляется в интерфейсе: радуется новым ачивкам, рекомендует контент, подбадривает.

Варианты использования:
- Лого и favicon
- Анимированные реакции на ачивки
- AI-гусь — персонаж и помощник в одном лице
- Стикерпак для Telegram
- Мерч потенциал

## Бизнес-модель: Open Core

### Открытое ядро (MIT License)
- Каталог всех медиа-типов
- Система провайдеров
- Система ачивок (статические)
- AI-рекомендации (BYOK — bring your own key)
- Self-hosted через Docker
- REST API
- Web UI

### Облачный сервис dostigus.ru (платный)
- Хостинг без заморочек
- Социальный слой (профили, лента, комменты)
- AI-рекомендации (включены, без своего ключа)
- AI-генерируемые ачивки
- Синхронизация между устройствами
- Приоритетная поддержка

### Техническая реализация

Один открытый репозиторий. Cloud-фичи отделены feature flag'ом `DOSTIGUS_CLOUD=true` и живут в отдельных Nuxt-модулях (`modules/cloud/`). Секреты — в K8s secrets / env vars, не в коде.

### Аналоги этой модели
- **Plausible** — open-source аналитика, платный облачный хостинг
- **Immich** — open-source альтернатива Google Photos
- **Umami** — open-source аналитика
- **Hoarder/KaraKeep** — open-source букмаркинг
